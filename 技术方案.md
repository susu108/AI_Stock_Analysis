# 多瑞生物(301075) 钉钉定时推送股票分析系统 — 技术方案

> **版本**: v1.0  
> **日期**: 2026-07-10  
> **目标**: 每天 A 股开盘前自动获取多瑞生物(301075)实时行情、K线、板块热点、资金流向等数据，综合分析当日涨跌趋势，给出买卖建议及理由，通过钉钉机器人推送到群。

## 一、系统概述

### 1.1 需求分析

| 需求项 | 说明 |
|--------|------|
| 定时推送 | 每个交易日开盘前（09:00，可配置）自动执行 |
| 实时数据 | 获取目标股票实时行情、近60日K线、资金流向 |
| 板块热点 | 获取概念板块和行业板块涨跌排名，分析市场情绪 |
| 涨跌预测 | 综合技术面、资金面、板块面三维度预测当日涨跌 |
| 买卖建议 | 给出买入价格区间、卖出价格区间、止损位及推荐理由 |
| 推送渠道 | 钉钉群机器人，Markdown 格式报告 |

### 1.2 技术选型

| 组件 | 技术 | 选型理由 |
|------|------|----------|
| 数据源 | akshare（东方财富系列接口） | 免费开源、无需注册、接口丰富、社区活跃 |
| 数据处理 | pandas + numpy | 金融数据处理的工业标准 |
| 定时调度 | schedule | 轻量级，无需数据库依赖，适合单机常驻 |
| 消息推送 | requests + 钉钉 Webhook | 钉钉官方推荐方式，支持 Markdown |
| 配置管理 | python-dotenv | 环境变量隔离，便于部署 |

## 四、数据采集层设计

### 4.1 接口清单

模块 `data_fetcher.py` 封装了 6 个 akshare 接口，所有函数均带 **3次重试机制**（`@_retry(max_retries=3, delay=2)`），失败返回 `None` 不中断主流程，每次成功调用后 `sleep(1)` 防反爬。

| 函数 | akshare 接口 | 参数 | 返回 |
|------|-------------|------|------|
| `get_realtime_quote()` | `stock_zh_a_spot_em()` | 无（全市场快照筛选） | dict：最新价、涨跌幅、成交量、换手率、量比、PE/PB 等 17 字段 |
| `get_kline(days=60)` | `stock_zh_a_hist()` | symbol, period="daily", adjust="qfq" | DataFrame：日期/OHLCV/振幅/涨跌幅/换手率，近60日 |
| `get_concept_boards()` | `stock_board_concept_name_em()` | 无 | DataFrame：概念板块涨跌排名（降序） |
| `get_industry_boards()` | `stock_board_industry_name_em()` | 无 | DataFrame：行业板块涨跌排名（降序） |
| `get_fund_flow()` | `stock_individual_fund_flow()` | stock, market="sz" | DataFrame：近100日主力/超大单/大单/中单/小单净流入 |
| `get_lhb(days=5)` | `stock_lhb_detail_em()` | start_date, end_date | DataFrame：龙虎榜明细（筛选目标股票） |

### 4.2 重试与容错机制

```python
@_retry(max_retries=3, delay=2)
def get_xxx():
    # akshare 调用...
```

- **重试策略**：失败后等待 2 秒重试，最多 3 次
- **降级处理**：3 次均失败返回 `None`，下游模块对 `None` 做容错处理
- **防反爬**：每次成功调用后 `time.sleep(1)`

### 4.3 数据汇总

`fetch_all_data()` 一次性调用所有采集函数，返回字典：

```python
{
    "realtime": dict | None,      # 实时行情
    "kline": DataFrame | None,    # K线数据
    "concept": DataFrame | None,  # 概念板块
    "industry": DataFrame | None, # 行业板块
    "fund_flow": DataFrame | None,# 资金流向
    "lhb": DataFrame | None,      # 龙虎榜（未上榜为None）
}
```

## 五、分析引擎设计

分析模块 `analyzer.py` 实现三维度独立评分，各维度评分范围 **-100 ~ +100**（正数看涨，负数看跌）。

### 5.1 技术面分析

基于近 60 日前复权 K线数据，计算以下技术指标：

#### 5.1.1 技术指标算法

| 指标 | 参数 | 计算公式 |
|------|------|----------|
| **MA** | 5/10/20/60 | `MA(n) = mean(close[-n:])` |
| **EMA** | 12/26/9 | `EMA[i] = price[i] * 2/(n+1) + EMA[i-1] * (1 - 2/(n+1))` |
| **MACD** | 12,26,9 | `DIF = EMA(12) - EMA(26)`；`DEA = EMA(DIF, 9)`；`MACD柱 = (DIF - DEA) * 2` |
| **RSI** | 6 | `RS = mean(gains) / mean(losses)`；`RSI = 100 - 100/(1+RS)` |
| **KDJ** | 9,3,3 | `RSV = (C - LLV(9)) / (HHV(9) - LLV(9)) * 100`；`K = EMA(RSV, 3)`；`D = EMA(K, 3)`；`J = 3K - 2D` |
| **布林带** | 20,2 | `中轨 = MA(20)`；`上轨 = MA + 2σ`；`下轨 = MA - 2σ`（σ为总体标准差） |
| **量价** | 10 | `量比 = 今日成交量 / 近10日均量` |

#### 5.1.2 技术面评分规则

| 信号 | 条件 | 评分 |
|------|------|------|
| 均线多头排列 | MA5 > MA10 > MA20 | +20 |
| 均线空头排列 | MA5 < MA10 < MA20 | -20 |
| 股价站上MA20 | price > MA20 | +10 |
| 股价跌破MA20 | price < MA20 | -10 |
| MA5金叉MA10 | 前日 MA5≤MA10，今日 MA5>MA10 | +15 |
| MA5死叉MA10 | 前日 MA5≥MA10，今日 MA5<MA10 | -15 |
| MACD金叉 | DIF上穿DEA | +20 |
| MACD死叉 | DIF下穿DEA | -20 |
| MACD红柱放大 | bar > prev_bar 且 bar > 0 | +10 |
| MACD绿柱放大 | bar < prev_bar 且 bar < 0 | -10 |
| MACD红柱（未放大） | bar > 0 | +5 |
| MACD绿柱（未放大） | bar < 0 | -5 |
| RSI超买 | RSI > 80 | -15 |
| RSI超卖 | RSI < 20 | +15 |
| RSI偏强 | 60 < RSI ≤ 80 | +5 |
| RSI偏弱 | 20 ≤ RSI < 40 | -5 |
| KDJ金叉 | K上穿D | +15 |
| KDJ死叉 | K下穿D | -15 |
| KDJ-J超买 | J > 100 | -10 |
| KDJ-J超卖 | J < 0 | +10 |
| 布林触及上轨 | position > 0.85 | -10 |
| 布林触及下轨 | position < 0.15 | +10 |
| 放量上涨 | 量比 > 1.5 且 涨 | +15 |
| 放量下跌 | 量比 > 1.5 且 跌 | -15 |
| 缩量上涨 | 量比 < 0.5 且 涨 | +5 |
| 缩量下跌 | 量比 < 0.5 且 跌 | -5 |

> 最终得分 `clip(-100, 100)`

### 5.2 资金面分析

基于个股资金流向数据（近5日），分析主力资金和机构资金动向：

| 信号 | 条件 | 评分 |
|------|------|------|
| 主力持续流入 | 近5日 ≥4天主力净流入为正 | +30 |
| 主力持续流出 | 近5日 ≥4天主力净流入为负 | -30 |
| 流入多于流出 | 正天数 > 负天数（非持续） | +10 |
| 流出多于流入 | 负天数 ≥ 正天数（非持续） | -10 |
| 最近1日主力净流入 | latest > 0 | +15 |
| 最近1日主力净流出 | latest < 0 | -15 |
| 超大单5日买入≥4天 | 机构资金持续进场 | +20 |
| 超大单5日买入≤1天 | 机构资金持续离场 | -20 |
| 累计净流入超千万 | 5日累计 > 1000万 | +10 |
| 累计净流出超千万 | 5日累计 < -1000万 | -10 |

### 5.3 板块面分析

基于概念板块和行业板块的涨跌数据，分析市场整体情绪：

| 信号 | 条件 | 评分 |
|------|------|------|
| 市场普涨 | 概念上涨占比 > 70% | +25 |
| 市场普跌 | 概念上涨占比 < 30% | -25 |
| 最强概念涨幅>3% | 热度较高 | +15 |
| 最弱概念跌幅>3% | 市场偏弱 | -15 |
| 领涨行业涨幅>2% | 行业景气 | +10 |
| 领跌行业跌幅>2% | 行业承压 | -10 |

### 5.4 综合预测模型

**加权公式**:

```
weighted_score = 技术面 × 0.40 + 资金面 × 0.35 + 板块面 × 0.25
```

> 权重设计逻辑：技术面反映价格趋势，权重最高；资金面反映主力意图，权重次之；板块面反映市场环境，作为辅助参考。

**方向判定**:

| 加权分数 | 方向 | 图标 |
|----------|------|------|
| > 25 | 看涨 | ⬆️ |
| < -25 | 看跌 | ⬇️ |
| 0 ~ 25 | 震荡偏涨 | ↗️ |
| -25 ~ 0 | 震荡偏跌 | ↘️ |
| = 0 | 震荡 | ➡️ |

**预测涨跌区间**:

```
est_change = weighted_score / 100 × 5      # 分数映射到 ±5% 区间
change_range = (est_change - 1.5%, est_change + 1.5%)
```

**信心指数（1~5星）**:

| |weighted_score| | 信心 |
|-------------------|------|
| ≥ 50 | ⭐⭐⭐⭐⭐ |
| ≥ 35 | ⭐⭐⭐⭐ |
| ≥ 20 | ⭐⭐⭐ |
| ≥ 10 | ⭐⭐ |
| < 10 | ⭐ |

## 六、买卖建议算法

模块 `advisor.py` 基于分析结果，通过支撑位和压力位计算买卖价格区间。

### 6.1 买入建议

#### 支撑位计算

| 支撑位 | 计算方法 | 说明 |
|--------|----------|------|
| **第一支撑位 S1** | `max(MA20, 布林中轨, 近20日最低价)` | 较强的支撑位，回调到此可首批买入 |
| **第二支撑位 S2** | `max(布林下轨, MA60, 近20日最低价)` | 更深支撑位，跌破则趋势可能反转 |

> 保证 S2 ≤ S1（若不满足则交换）

#### 买入参数

- **买入区间**: S2 ~ S1
- **止损位**: `S2 × 0.98`（第二支撑位下方 2%）
- **建议买入条件**: `技术面评分 + 资金面评分 > 0`（显示🟢，否则显示🟡观望）

### 6.2 卖出建议

#### 压力位计算

| 压力位 | 计算方法 | 说明 |
|--------|----------|------|
| **第一压力位 R1** | `min(近20日最高价, MA5, 布林上轨)` | 最先遇到的阻力位 |
| **第二压力位 R2** | `max(布林上轨, 近20日最高价, ceil(R2_base)整数关口)` | 更强阻力位 |

> 保证 R2 ≥ R1

#### 卖出参数

- **卖出区间**: R1 ~ R2
- **建议卖出条件**: `技术面评分 + 资金面评分 < 0`（显示🔴，否则显示🟡持有）

### 6.3 操作策略生成

基于综合加权分数自动生成文字策略建议：

| 加权分数 | 操作策略 |
|----------|----------|
| > 30 | 整体偏强，可逢回调至支撑位分批低吸，设好止损 |
| 10 ~ 30 | 温和偏多，轻仓试探性买入，关注量能配合 |
| -10 ~ 10 | 方向不明，建议观望为主，等待趋势明确 |
| -30 ~ -10 | 温和偏空，持仓者可逢反弹至压力位减仓 |
| < -30 | 整体偏弱，建议规避或轻仓，不宜追涨 |

### 6.4 推荐理由生成

从技术面和资金面信号中自动提取关键依据：

- **买入依据**：提取含"金叉""多头排列""超卖""支撑""放量上涨""站上MA20"等关键词的信号，最多3条
- **卖出依据**：提取含"死叉""空头排列""超买""压力""放量下跌""绿柱"等关键词的信号，最多3条

## 七、钉钉推送设计

### 7.1 加签算法

```python
timestamp = str(round(time.time() * 1000))           # 1. 毫秒时间戳
string_to_sign = f"{timestamp}\n{secret}"              # 2. 拼接签名字符串
hmac_code = hmac.new(                                  # 3. HMAC-SHA256
    secret.encode("utf-8"),
    string_to_sign.encode("utf-8"),
    digestmod=hashlib.sha256
).digest()
sign = urllib.parse.quote_plus(base64.b64encode(hmac_code))  # 4. Base64 + URL编码
# 5. 拼接到 URL: {webhook}&timestamp={ts}&sign={sign}
```

### 7.2 消息格式

- **类型**: `markdown`
- **Payload**: `{"msgtype": "markdown", "markdown": {"title": title, "text": text}}`
- **HTTP**: `POST`，超时 10 秒
- **频率限制**: 钉钉每分钟限 20 条，每日1条不触发

### 7.3 推送报告结构

```markdown
# 📊 多瑞生物(301075) 每日分析报告
> 2026-07-10 09:00 ｜ 开盘前参考

## 一、实时行情
（最新价、涨跌幅、今开/昨收、最高/最低、成交量/额、换手率、量比、PE/PB）

## 二、综合预测
（方向+图标 ｜ 信心星级 ｜ 涨跌区间 ｜ 三维评分明细）

## 三、买卖建议
（🟢买入区间 + 支撑位 + 止损位 ｜ 🔴卖出区间 + 压力位 ｜ 操作策略）

## 四、推荐理由
（买入依据 2-3条 ｜ 卖出依据 2-3条）

## 五、技术面信号
（均线/MACD/RSI/KDJ/布林带/量价 信号列表）

## 六、资金面信号
（主力净流入趋势/超大单动向/累计金额）

## 七、市场热点
（概念板块涨跌数 ｜ TOP5概念排名）

## ⚡ 龙虎榜信号（仅当上榜时展示）

> ⚠️ 以上分析由程序自动生成，仅供参考，不构成投资建议。股市有风险，投资需谨慎。
```

### 7.4 推送容错

| 场景 | 处理 |
|------|------|
| Webhook 未配置 | 跳过推送，报告输出到日志 |
| 网络异常 | 捕获异常，记录错误日志 |
| errcode ≠ 0 | 记录钉钉返回的错误信息 |
| errcode = 0 | 推送成功 |

## 八、调度与运维设计

### 8.1 定时调度

```python
# 手动模式（测试）
if MANUAL_RUN:
    job()  # 立即执行一次

# 定时模式（生产）
else:
    schedule.every().day.at("09:00").do(job)
    while True:
        schedule.run_pending()
        time.sleep(30)  # 30秒轮询
```

### 8.2 交易日判断

- **周末排除**: `weekday() >= 5` → 跳过
- **节假日排除**: 内置 2025-2026 年 A 股休市日（元旦、春节、清明、劳动节、端午、中秋、国庆）
- 非交易日仅记录日志，不执行分析

### 8.3 异常降级策略

| 场景 | 降级处理 |
|------|----------|
| 单个数据源失败 | 重试3次后返回 `None`，不中断主流程 |
| K线+实时行情均失败 | 推送"数据异常"通知（不分析），通知用户检查网络 |
| K线正常但其他数据缺失 | 对应维度评分返回 0，signals 标注"数据不足"，继续执行 |
| 钉钉推送失败 | 记录警告日志，不重试 |
| 非交易日 | 跳过整个任务 |

### 8.4 部署方式

| 方式 | 命令 | 适用场景 |
|------|------|----------|
| **nohup** | `nohup python3 main.py > app.log 2>&1 &` | 快速部署 |
| **screen** | `screen -S stock` → `python3 main.py` → `Ctrl+A,D` | 交互调试 |
| **systemd** | 见下方配置 | 生产环境（推荐） |

systemd 服务配置 `/etc/systemd/system/stock-analyzer.service`:

```ini
[Unit]
Description=Stock Analyzer DingTalk Pusher
After=network.target

[Service]
Type=simple
WorkingDirectory=/path/to/stock-analyzer
ExecStart=/usr/bin/python3 main.py
Restart=always
RestartSec=60

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable stock-analyzer
sudo systemctl start stock-analyzer
```
